**Gestión del proyecto**

Herramienta utilizada:
  Para la gestión del proyecto se ha utilizado **Trello**, una herramienta basada en la metodología Kanban que permite organizar tareas mediante tableros visuales.

Organización del tablero
  El tablero se ha dividido en varias columnas para representar el estado de las tareas:

  Backlog: contiene todas las ideas y funcionalidades pendientes
  Todo: tareas listas para comenzar 
  In Progress: tareas en desarrollo
  Review: tareas finalizadas pendientes de revisión
  Done: tareas completadas
  
Gestión de tareas
  Cada funcionalidad del proyecto se ha representado como una tarjeta dentro del tablero. A su vez, cada tarjeta contiene una lista de subtareas técnicas que permiten dividir el trabajo en partes más pequeñas y manejables.
  
  Esto facilita la organización y permite tener un mayor control sobre el progreso del desarrollo.

Flujo de trabajo
  Las tareas se van moviendo entre columnas según su estado:

  1- Se añaden al Backlog
  2- Se seleccionan y pasan a Todo
  3- Cuando se empiezan, pasan a In Progress
  4- Una vez terminadas, pasan a Review
  5- Finalmente, tras revisión, se mueven a Done

Organización del trabajo
  El trabajo se organiza de forma progresiva, priorizando primero las funcionalidades principales del sistema, como el calendario y la gestión de eventos.
  
  Posteriormente, se añadirán funcionalidades opcionales y mejoras.
